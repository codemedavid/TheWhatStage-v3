Property Page CTA FAB — Design

Section 1 of 3 — Architecture & Component Structure

New file: CTAFab.client.tsx

One new client component alongside the existing Renderer.tsx in the realestate kind folder. PropertyDetail (server component) instantiates it and passes:

  • pages — lightweight metadata array { id, kind, title, cta_label } for rendering the FAB buttons
  • accent — theme accent color for button styling
  • children — pre-rendered LinkedRenderer outputs (one per linked page), passed as React nodes so they render on the server, not the client

What changes in Renderer.tsx

Only PropertyDetail is modified — wrap the existing LinkedRenderer calls and pass them as children to CTAFab. The existing "Next Steps" inline section stays untouched.

Component tree

PropertyDetail (server)
  └─ CTAFab.client (client)
      ├─ FabButton (expands/collapses menu)
      ├─ FabMenu (list of page buttons)
      └─ Sheet (bottom sheet / modal)
          └─ children[activeIndex] (server-rendered form/booking)
