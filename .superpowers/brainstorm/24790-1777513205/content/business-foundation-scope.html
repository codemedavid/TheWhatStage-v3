What should the first foundation deliver?

The goal is to create a secure shared base that products, properties, digital products, and future services can build on without duplicating data or overloading RAG.

A. Shared Core Only
My Business
Product schema
Property schema
Digital schema
Media + RAG source contracts
No full customer storefront yet

A. Shared Core Only

Build database, permissions, item model, media rules, and RAG/action-page interfaces first. Fastest secure base, but less visible product value in the first pass.

B. Core + Product Catalog
My Business / Products
Product editor
Product card preview
RAG can retrieve one product at a time
Catalog action page reads products

B. Core + Product Catalog

Build the shared foundation and make e-commerce products usable first. Best balance because products prove the RAG and action-page model.

C. Core + All Store Types
My Business
Products
Properties
Digital products
Services draft
Multiple public pages at once

C. Core + All Store Types

More complete up front, but higher security and UX risk. This is likely too large for one clean implementation pass.
