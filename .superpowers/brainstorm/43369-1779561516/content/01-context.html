Echo system — what exists today

Where confirmation messages come from on submission, and where templates would slot in

Echo resolution chain (today)
1. outcomeAction.messenger_text  (qualification per-outcome card)
↳ first non-empty wins
2. pipeline_rules[outcome].notify_text  (per-outcome editor)
↳ first non-empty wins
3. page.notification_template.text  (global page echo)
→ for catalog: buildOrderEcho() auto-prepends order summary + customer block, then appends the resolved text.
→ send as { kind: 'text' } via sendOutbound

What each page kind has on submission

Booking

Data: slot_iso, fields (full_name, phone, +custom)
Echo today: plain text from notification_template.text
Outcomes: booked / invalid

Catalog (products)

Data: items[], customer{}, payment_method_id, payment_proof_url
Echo today: auto order-summary block + appended notify text
Outcomes: checked_out

Sales

Data: customer{}, optional payment_method_id, payment_proof_url
Echo today: plain text from notification_template.text
Outcomes: submitted / payment_submitted

Property (realestate)

Data: raw payload — currently passes through untyped
Echo today: plain text from notification_template.text
Outcomes: inquiry_submitted / viewing_booked
